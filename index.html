Prepare the file
📥
Download the template
Always start from the Port Store template. Column names must match exactly — don't rename or add columns.
📥  Download Lightspeed Upload Template
📄
One row per variant
Each size gets its own row. Product name, category, prices, brand, and supplier repeat on every row — only the SKU, barcode, and size value change per row.
ℹ️
Products with no variants (e.g. a can cooler) still get one row — leave variant_option_one_name and variant_option_one_value blank.

📝Product names are identical across all rows for the same product
🏷️Tags are semicolon-separated with no spaces: 2026 Macron Range;2026 SDNR

Stage 3
Run the import
⚠️
Test first. Run a small file of 5–10 products before your full upload.
🖥️
Navigate to the import tool
Products›Import / Export›Import Products
☑️
Select import type
Choose Add new products for a new range. Only use Update existing products if changing fields on products already in Lightspeed.
🚫
Don't use "Update existing" for a new range — rows with no existing match are silently skipped.
📤
Upload CSV and confirm mapping
Click Choose File, select your CSV, then Next. Review every field and manually assign anything showing "Not mapped".
💡
Screenshot your mapping before confirming — useful if you import the same product type regularly.
▶️
Run the import
Click Import and don't navigate away. When complete, check the skipped count — any skips above zero need investigating.
After the import — quick checks
🔍
Spot check productsSearch 2–3 products and confirm name, SKU, price, and category are correct.
📐
Check variantsOpen one sized product and confirm all sizes are present.
📱
POS scan testScan a barcode at the terminal — correct product and price should appear.
💾
Save the import fileArchive your CSV: YYYYMMDD_RangeName_import.csv
Stage 4
Adding barcodes after the initial upload
ℹ️
If barcodes weren't available at upload time, add them later using a tag-filtered export and re-import.
🏷️
Filter by tag in Back Office
Go to Products and filter by the range tag you added during the initial upload (e.g. 2026 Macron Range).
📤
Export the filtered product list
With the tag filter active, go to Import / Export and export. This gives you a CSV of only those products with all existing fields populated.
💡
Use sku_code_custom and name to match your barcode data against the right products.
✏️
Add barcodes to the exported CSV
Fill in the sku_code_barcode column. Format as Text before entering values.
⚠️
Only edit the sku_code_barcode column. Changing other fields will overwrite existing data.
🔢Column formatted as Text before entering values
🔢All barcodes are 13 digits (EAN-13)
🔢No duplicate barcodes in the file
💾Save as CSV UTF-8 before re-importing
▶️
Re-import using "Update existing products"
Go to Products → Import / Export → Import Products and select Update existing products. Lightspeed matches rows by sku_code_custom.
💡
This is the correct time to use Update existing products — the products already exist and you're only filling in barcode data.
📱
Verify with a POS scan test
Scan one of the newly added barcodes at the POS terminal to confirm it pulls up the correct product and price.
Troubleshooting
⚠️ Rows are being skipped with no error
Usually a column mapping issue or wrong import type. Check headers match the template exactly, and confirm you chose Add new products.
🔢 Barcodes show as scientific notation (8.05E+12)
The barcode column is formatted as a number. Select the column, format as Text, re-paste the barcodes as values, then re-export the CSV.
📄 Duplicate products instead of variants grouped together
A variant row has a slightly different product name — even a trailing space causes Lightspeed to create a new product. Use Excel find-replace to standardise the name.
🗂️ Category not found / product has no category
The category name doesn't exactly match Lightspeed. Pull the exact name from Products → Categories in Back Office.
💰 Prices show as $0.00 after import
Price columns contain a $ symbol or comma separator. Remove with Excel find-replace, re-export, then re-import using Update existing products.
🏢 Supplier error on import
The supplier must already exist in Lightspeed. Go to Inventory → Suppliers and add it first. Supplier names are case-sensitive.

Stage 1
Get your data
There are two ways to source product data for an AWDS import. Which one you use depends on whether you're working from a new range upload or from products already in Lightspeed.
📊
From the product spreadsheet
Recommended
Use the pre-built product upload spreadsheet for the range. This already has SKUs, descriptions, categories, and barcodes in one place.
Barcodes are already present
Data is clean and consistently formatted
Much faster for large ranges
🖥️
From a Lightspeed export
Export products from Lightspeed Back Office. Only use this when you don't have the original upload spreadsheet.
Useful for products already live in Lightspeed
Lightspeed does not export barcodes — you'll need to source them separately.
⚠️
Lightspeed barcode limitation
Lightspeed's export deliberately omits barcodes. For any range larger than a handful of products, use the product spreadsheet path wherever possible.
